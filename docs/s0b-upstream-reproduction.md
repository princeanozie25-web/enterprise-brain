# Microsoft Entra Agent ID preview — S0b upstream reproduction

## Summary

An authenticated, personally administered Microsoft Entra tenant has a genuine
Agent Identity linked to a valid agent blueprint. The supported
`Microsoft.Identity.Web.AgentIdentities` autonomous flow succeeds but returns
an app-only token without the Agent Identity facets documented for that flow.

Enterprise Brain correctly rejects that token because the signed evidence that
the subject and actor are Agent Identities is absent.

## Tenant and objects

| Item | Value |
| --- | --- |
| Tenant type | Microsoft Entra ID tenant, personally administered |
| Entra licence tier | Not captured by the diagnostic; provide to Microsoft support from the tenant overview |
| Tenant ID | `<tenant-id>` |
| Agent Identity service-principal object/client ID | `<agent-identity-client-id>` |
| Agent Identity status | `genuine_agent_identity = true`; `agentIdentityBlueprintId` present |
| eb-gateway application (resource) client ID | `<gateway-app-id>` |
| Resource audience | `api://<gateway-app-id>` |
| App role | `Agent.Access`, directly assigned to the Agent Identity |

The blueprint identifier is intentionally omitted here: it is safe metadata,
but should be supplied to Microsoft through the tenant administrator's support
channel together with the Graph diagnostic output.

## SDK and host

| Component | Value |
| --- | --- |
| NuGet package | `Microsoft.Identity.Web.AgentIdentities` `4.13.0` |
| NuGet package | `Microsoft.Identity.Web` `4.13.0` (resolved transitively with the minter) |
| Minter target framework | `net8.0` |
| Installed .NET SDK | `10.0.300` |
| PowerShell | `5.1.26100.8655` |

The diagnostic/minter is generated by
[`tools/s0b-agent-id-diagnostics.ps1`](../tools/s0b-agent-id-diagnostics.ps1).
It holds the blueprint credential and token in memory only, emits only the
safe claim subset below, and invokes the live bridge test with the token passed
only through its child-process environment.

## Supported SDK call pattern

```csharp
builder.Services.AddMicrosoftIdentityWebApiAuthentication(builder.Configuration);
builder.Services.AddAgentIdentities();
builder.Services.AddInMemoryTokenCaches();

var provider = app.Services.GetRequiredService<IAuthorizationHeaderProvider>();
var options = new AuthorizationHeaderProviderOptions()
    .WithAgentIdentity("<agent-identity-client-id>");
var header = await provider.CreateAuthorizationHeaderForAppAsync(
    "api://<gateway-app-id>/.default", options);
```

`AzureAd:ClientId` is the **blueprint** client ID. The value passed to
`WithAgentIdentity` is the Agent Identity client ID above.

## Expected versus actual

Microsoft's Agent ID token-claims reference documents autonomous agent tokens
with `idtyp: app`, an application role, and both `xms_sub_fct: 11` and
`xms_act_fct: 11`; `xms_par_app_azp` is an audit attribution claim when
available. The result above has the role and Agent Identity identifiers but
lacks all Agent Identity marker claims.

## Minimal reproduction

1. Create an Agent Identity and record its client ID.
2. Link it to an agent blueprint with a valid client credential.
3. Create the `Agent.Access` app-role assignment on the resource application.
4. Generate the local minter with `tools/s0b-agent-id-diagnostics.ps1` and
   supply the blueprint credential only at its secure prompt.
5. Request the resource scope shown above through
   `WithAgentIdentity(agentIdentityClientId)`.
6. Decode only the safe claim set and observe that both facet claims are absent.

## Gateway behaviour

The Enterprise Brain bridge validates the Entra signature, version-appropriate
issuer, audience, expiry, and tenant before it classifies identity shape. A
token without `idtyp: app` and without both signed AgentIdentity facets is not
accepted as autonomous. It is denied with `agent_facets_missing` (the
attestation evidence is absent — the shape is not provably delegated either);
it never reaches the Agent registry or any resource authorization path.
Allowing it would permit a regular app-only token to impersonate an Agent
Identity based only on matching identifiers/role.
